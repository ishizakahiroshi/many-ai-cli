# [完了] 障害対応記録: Codexスピナー「•」がチャット履歴に大量蓄積される

> 最終更新: 2026-05-17(日) 23:39:24

## 症状

Codex セッションで大きな CSS ファイル（`web/src/styles.css`）を分割読み込みするなど、
長めのツール実行を行うと、ユーザーとほぼやり取りしていないにもかかわらずチャット件数が
97件（AI出力タブ 94件）まで膨らむ。

- Chat ペインに「•」だけのエントリが 1秒おきに連続して積まれる
- 実際の会話内容は入力 1件・承認 1件・画像 1件のみ
- CLI（ターミナル直接実行）では同じ操作でこの現象は起きない

**追加症状（フォローアップ）**:
スピナーフィルター追加後も、承認プロンプト表示中に「•」だけのエントリが
チャット履歴に蓄積し続ける場合がある。
Codex が `• ` のように末尾にスペースを付けて出力するケースで再発する。

## 根本原因（root cause）

Codex CLI はツール実行中にスピナーアニメーションとして「•」を出力する。
ターミナル上では以下のようなシーケンスでインプレース上書きするため、蓄積されない。

```
\r\x1b[K•   ← キャリッジリターン + 行消去 + 「•」を描画（毎回同じ位置）
```

any-ai-cli では PTY 生データを WebSocket 経由でブラウザに送り、
`chatHistoryAppendOutput`（`web/src/app.js:1630`）でバッファリングし、
800 ms 静止後に `chatHistoryCommitOutput`（同:1643）で `chatHistory` に push する。

`stripAnsiBasic`（同:1528）は ANSI エスケープを除去するが「•」（U+2022）は
制御文字でも ANSI シーケンスでもないため残る。静止タイマーは Codex のスピナーが
ちょうど ~1 秒間隔で出力されることで毎回発火し、「•」だけのエントリが
セッション中ずっと push され続けた。

既存の空白チェック（`if (!stripped.trim()) return;`）も「•」は
`trim()` 後に空にならないため通過してしまっていた。

## 修正内容

`chatHistoryCommitOutput` に spinner 文字のみのチャンクを弾くガードを追加。

**before:**
```javascript
const stripped = stripAnsiBasic(raw);
if (!stripped.trim()) return;
// 起動バナー混入防止...
```

**after:**
```javascript
const stripped = stripAnsiBasic(raw);
if (!stripped.trim()) return;
// spinner / progress 文字のみのチャンクは無視（例: Codex が出力する "•"）
if (/^[\r\n•◦⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏]+$/.test(stripped)) return;
// 起動バナー混入防止...
```

正規表現は「•」「◦」に加え、ブレイルスピナー（`⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏`）や
改行・CR のみの組み合わせも対象にしている。

**フォローアップ修正（2026-05-17）**: 初回フィルターは `stripped`（生テキスト）に対して
正規表現を適用していたため、`• `（末尾スペース付き）の場合にスペースが正規表現の文字クラスに
含まれず通り抜けていた。`stripped.trim()` に変更して前後の空白・タブを除いた上で判定するよう修正。

**after（フォローアップ）:**
```javascript
const stripped = stripAnsiBasic(raw);
if (!stripped.trim()) return;
// spinner / progress 文字のみのチャンクは無視（例: Codex が出力する "•"）
// stripped.trim() で前後の空白・タブを除いた上で判定（"• " など末尾スペース付きを取りこぼさないため）
if (/^[•◦⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏\r\n]+$/.test(stripped.trim())) return;
```

## 備忘

CLI で見えない理由は「ANSI シーケンスによるインプレース上書き」であり、
スピナー文字自体は PTY に流れている。他のターミナル出力をテキスト変換する箇所でも
同様の問題が起きる可能性がある（`stripAnsiBasic` だけでは描画意図まで除去できない）。

また Codex 以外の CLI ツールが同種のスピナーを使っている場合も本修正が有効に機能する。
