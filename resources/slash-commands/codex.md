# Codex Slash Commands

| Command | Purpose | When to use it |
|---|---|---|
| `/permissions` | Set what Codex can do without asking first. | Relax or tighten approval requirements mid-session. |
| `/sandbox-add-read-dir` | Grant sandbox read access to an extra directory. | Unblock reads for a directory outside the current readable roots. |
| `/agent` | Switch the active agent thread. | Inspect or continue work in a spawned subagent thread. |
| `/apps` | Browse apps and insert them into your prompt. | Attach an app before asking Codex to use it. |
| `/plugins` | Browse installed and discovered plugins. | Enable, inspect, or use plugin-provided capabilities. |
| `/skills` | Browse available skills. | Inspect bundled or installed skills for the current task. |
| `/model` | Choose the active model. | Switch models or reasoning effort before running a task. |
| `/fast` | Toggle fast mode. | Prefer lower-latency responses when deep reasoning is not needed. |
| `/personality` | Change the response style. | Adjust how Codex communicates in the session. |
| `/status` | Open status and settings information. | Check version, model, account, connectivity, and active settings. |
| `/clear` | Clear the current conversation context. | Start fresh without leaving the session. |
| `/compact` | Summarize and compact the conversation. | Free context while preserving important state. |
| `/resume` | Resume a previous conversation. | Continue an earlier session. |
| `/help` | Show help. | Find available commands and usage hints. |
